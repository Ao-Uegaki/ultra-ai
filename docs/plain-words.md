# plain-words — 言い換え表(わかりやすい言葉づかいガイド)

## この文書は誰のため・何のため

- **読む人**: ultra-ai に手を入れる人(あなた / 将来のあなた / 貢献者)と、文言を書く Claude セッション。
- **何のため**: 人が読む文言(README・CLAUDE.md・hook のメッセージ・コメント・docstring)を**辞書なしで読める**ようにする。
  難しい語を見つけたら、ここの**同じ置き換えを全箇所で一貫**して使う(場所ごとに訳が割れないように)。
- この表は「正本」= 迷ったらここに合わせる。可視(git で見られる)で、いつでも直せる。

専門語のメモ: **jargon**(ジャーゴン=その分野の人にしか通じない用語)、**byte-stable**(バイト単位で毎回まったく同じ文字列。
1 文字でも変わると prompt cache〔毎回同じ前置きの計算を使い回す仕組み〕が効かなくなる)、**enum**(取りうる値が決まった列挙)、
**verbatim**(一字一句そのまま)。これらは下で出てくるので先に注釈しておく。

## 名前の方針(ファイル名・概念名)

- 人が読むファイル名・概念名に、**辞書なしで読めない英単語を使わない**(例: `glossary` → `plain-words`、`harness` → 「ultra-ai の仕組み」)。
- ただし**日本語ファイル名にはしない**。読みやすい簡単な英語にする。
- 例外: Claude Code 由来の固定語(`hooks`/`skills`/`agents`/`settings.json`)や一般的な開発語(`bench`/`tests`)はそのまま。

## 残す(厳密な技術用語 — 置換しない)

これらは**正確な意味を持つ標準語**なので置き換えない(読者を技術者と想定)。

- 日: `冪等`(何度実行しても同じ結果)・`単一障害点`(そこが壊れると全体が止まる箇所)・`関門`(条件を満たすまで先へ進ませない仕組み)・
  `常駐`(毎セッション必ず読み込ませる)・`委譲`・`反映`/`記録`(学習の中核概念)
- 英: `idempotent`・`provenance`(出どころ)・`byte-stable`・`held-out`/`oracle`(採点用に隠しておくテスト)・
  `fan-out`/`solo`(enum)・`heuristic`・`compaction`(会話圧縮)・`stale`/`closure`/`state mutation`(言語の標準語)

### 例外: README 等「入口」では、残す語にも初出 1 回だけ短い注釈(語は残す)

README は公開リポの入口で、初めて読む人が詰まらないように。注釈は**初出 1 回**だけ・以降は素の語。

- `symlink`(別名のリンク/ショートカット)
- `PATH`(コマンド検索パス) / `macOS Keychain`(macOS の資格情報ストア) / `tomllib`(Python 標準の TOML 読み取り)
- `autofix`(自動修正) / `runtime state`(実行時に溜まるデータ) / `transcript`(セッションの会話ログ) / `marker`(目印の囲みコメント)
- `冪等`(何度実行しても同じ結果)

## 対象外(verbatim 厳守・絶対に変えない)

これは「読みやすさ」より**正しく動くこと**が優先だから。1 文字変えると壊れる/意味がズレる:

- 機械語キー(JSON の key)・enum 値(`PASS`/`FAIL`/`UNKNOWN`・`fan-out`/`solo`・`control`/`treatment`)
- 指標キー(`cost_weighted_total`/`pass_rate`/`peak_main_context` 等)
- 環境変数(`UA_*`)・コマンド名(`ultra-ai`/`uai`/`/ua-*`)・ファイル名/hook 種別(`gate.py`[Stop] 等)
- approach の戦略キーワード・コミットメッセージ

## 原則

- **規則の効力を弱めない**: 「〜しない / 必ず〜する / 〜だけ可」等の強さ・条件・例外を、語彙の言い換えで薄めない。
- **byte-stable 文言**(CLAUDE.md / approach 足場 / 注入ヘッダ)は**語彙だけ平易化し、意味は 1 バイトも劣化させない**。
  変更後の文言が、新しい byte-stable の正本になる。
