name: ua-failpass
description: 非自明な挙動変更を、確定 FAIL のテストを先に書いてから実装で PASS にする(実行可能な受け入れ条件)。ua-spec の acceptance を実行可能化。些細な変更には使わない。
---

非自明な挙動変更(新機能・バグ修正・仕様変更)を **FAIL→PASS** で進める。PASS になるまで直す繰り返しを
「実行可能な受け入れ条件」で裏打ちし、テスト無しの空虚な PASS を防ぐ。

手順:
1. **受け入れ条件を 1〜数個の具体例に**(`/ua-spec` を使ったならその acceptance をそのまま使う)。
2. **失敗するテストを先に書く** → 実行して **確定 FAIL** を確認する(まだ実装していないので落ちる=テストが本当に
   挙動を見ている証拠)。落ちなければテストが弱い → 直す。
3. **最小実装で PASS にする**。`verify.py`[PostToolUse]/ `gate.py`[Stop]の PASS の関門(条件を満たすまで先へ進ませない仕組み)が自動で回る。PASS まで反復。
4. リファクタは PASS を保ったまま(必要なら `/ua-refactor`)。

注意:
- **些細な変更(typo・1行・自明)には使わない**(オーバーヘッドが価値を上回る)。
- 確定 **FAIL → PASS** を通っていない変更は「未検証」。`UNKNOWN`(テスト未検出)は PASS ではない。
- 既存のテスト規約に合わせる(このリポなら `python3 -m unittest discover -s tests`)。
