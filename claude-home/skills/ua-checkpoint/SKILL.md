name: ua-checkpoint
description: 現在の(PASS の)状態を安全にチェックポイント・コミットする。追跡済み変更のみコミットし、未追跡=機密は含めない。明示的に頼まれたときだけ使う。
---

現在の作業を安全にチェックポイント・コミットする。

手順:
1. Bash で `python3 "$CLAUDE_CONFIG_DIR/hooks/checkpoint.py"` を実行する。
   - 追跡済みの変更だけを `git add -u` でステージしてコミットする(未追跡ファイルは含めない=機密混入を防ぐ)。現ブランチにコミットし、メッセージは決定論的に生成される。
2. 新規ファイルもコミットに含めたい場合は、**機密でないことを確認**してから明示的に `git add <path>` し、再度上記を実行する。
3. 結果(対象ファイル数・ブランチ・コミットハッシュ)を簡潔に報告する。`main`/`master` への直コミット警告が出たら伝える。

注意: コミット失敗は検証失敗とは別物。失敗時はそのまま報告し、PASS になるまで直す繰り返しとは混同しない。
