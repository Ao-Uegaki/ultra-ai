## バックエンド(API/サーバ・Python/Node)の定石

- 入力は境界で検証(型+値)。信頼しない。SQL/コマンドはパラメータ化(文字列結合しない)。
- エラーは握りつぶさず、呼び出し側に意味のある形で返す(適切な HTTP ステータス+機械可読なコード)。
  内部詳細・スタックを外に漏らさない。
- 何度実行しても結果が同じ(冪等)にする工夫と再試行:書き込み API は冪等キー/楽観ロックを検討。外部呼び出しは timeout + リトライ(指数)。
- トランザクション境界を明確に。1件ずつ何度も問い合わせる無駄(N+1)を避け、必要な分だけ取得する。
- 秘密情報は環境変数/シークレットストアから。コード・ログに出さない。個人を特定できる情報(PII)はログに残さない。
- 観測可能性:構造化ログ(stderr)、相関 ID、主要操作にメトリクス。
- 後方互換:API 変更はバージョン/段階移行。破壊的変更を黙って入れない。
