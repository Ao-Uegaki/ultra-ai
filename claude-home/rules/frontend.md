## フロントエンド(React/TS)の定石

- コンポーネントは小さく単責務。表示(presentational)とロジック/データ取得を分ける。
- 状態は最小・近接に置く。サーバ状態(取得データ)はクライアント状態と分けて扱う(React Query 等)。
  派生値は state にせず計算する。
- `useEffect` は最後の手段(同期の逃げ道)。依存配列を正しく、副作用は1目的ずつ。
- key は安定 ID(index を避ける)。リスト/フォームの制御を明確に。
- アクセシビリティ:意味のある HTML 要素、ラベルと入力の関連付け、フォーカス可視、キーボード操作可。
- パフォーマンス:不要な再レンダリングを memo 前に計測で確かめる(早すぎる最適化をしない)。
- 例外/ローディング/空 の3状態を必ず設計する(成功パスだけにしない)。
