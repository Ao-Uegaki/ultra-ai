## Python の約束ごと(一般則)

- 型注釈を付ける(公開関数・戻り値)。`from __future__ import annotations` で前方参照を簡潔に。
- 例外は握りつぶさない。捕まえるなら具体的な型で。境界(hook/IO)だけは広く捕まえ、エラーを受け止めて安全にあきらめてよい(プログラムは終了させず先へ進む。hook なら何もせず素通り)。
- 標準ライブラリで足りるなら依存を足さない。`pathlib` でパス、`subprocess.run([...])`(`shell=True` は信頼境界でだけ)。
- 副作用と純ロジックを分ける(純関数はテストしやすい)。I/O は薄く、判断は純関数に。
- フォーマットは ruff/black 既定に従う。早期 return でネストを浅く。
- print デバッグは残さない。ログは stderr / logging へ。
- テストは `python3 -m unittest`(このリポ規約)。assert は1観点ずつ、名前で意図を表す。
